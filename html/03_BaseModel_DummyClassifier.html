# Function to evaluate the model
def evaluate_model(ytest, ypred, ypred_proba = None):
    if ypred_proba is not None:
        print('ROC-AUC score of the model: {}'.format(roc_auc_score(ytest, ypred_proba[:, 1])))
    print('Accuracy of the model: {}\n'.format(accuracy_score(ytest, ypred)))
    print('Classification report: \n{}\n'.format(classification_report(ytest, ypred)))

2. Creación del modelo.¶

Vamos a utilizar el modelo DummyClassifier como modelo base, estableciendo que la estrategia sea most_frequent, es decir, su prediccin ser exclusivamente la clase que ms se repite.

  • Support nos dice los datos reales que tenemos, 43264 que son 0 y 110766 que son 1.
  • La precisin se calcula con las columnas de la matriz de confusin y su precisin es nula (0) para los casos 0 y 0.72 para los casos 1. Acierta todas las puntuaciones altas (1) pero falla todas las bajas (0) al predecirlas como altas. La precisin es alta, pero debido a que la variable de destino est desequilibrada y la mayora son 1.
  • El recall se calcula con las filas de la matriz de confusin, la prediccin es mala porque no logra identificar las puntuaciones bajas, pero acierta el 100% de las puntuaciones altas.
  • El f1 es la combinacin de estas dos mtricas, para tener una ms exacta. Identifica 0.84 de puntuacin alta, pero no identifica ninguna puntuacin baja.
  • El accuracy se calcula con la matriz de confusin, teniendo en cuanto los aciertos y es 0.72, es decir, que el modelo predice correctamente el 72% de los casos, pero esto no quiere decir que sea un buen modelo aunque su exactitud pueda parecer alta, como ya se ha visto con el resto de mtricas, las predicciones no son buenas, este resultado se debe al desequilibrio del target.
